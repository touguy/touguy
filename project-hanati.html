Project Overview

This summary covers my major projects in this company. which were primarily for financial companies.

[Hana Financial Group Cloud Strategy & Action Plan Design] (Hana Financial Group, ’21. 11 ~ Now)
- Project Goal: Establishment of Group Cloud Advancement System for Digital Innovation of Hana Financial Group
- Responsibilities: Open-source TA (customer service support)
   (1) Establishing cloud governance & standard processes & defining architectural requirements
   (2) Designing hybrid cloud standard usage processes and related infrastructure
   (3) Defining & designing Group Cloud Native standard platform requirements, POC support

[Hana Insurance System Transformation] (Hana Insurance, ’21. 02 ~ ’21. 11)
- Project Goal: Hana insurance(舊TheK insurance) Insurance Business System Transformation(Rebuild for Next Generation)
- Responsibilities: AA/SWA (customer service support)
   (1) Defining the Application Architecture Requirements for Insurance Business Systems
   (2) Assists in reviewing and configuring various deliverables related to application architecture design/implementation
   (3) Identify issues related to configuring response development architecture and support decision making to resolve issues

[Hana Capital Installment Rental System Integration] (Hana Capital, ’20. 12 ~ ’21. 02)
- Project Goal: Integrate capital installment system to private cloud (HanaCloudia) and related open source design and technical support for Hana Capital
- Responsibilities: Design and technical support for open source solution
   (1) Defining Architecture Requirements and Configuration Definitions for Capital Installment Rental Platform Infrastructure and Application
   (2) Support for cloud transition configuration and open source (CentOS, Nginx, MariaDB) configuration technology

[Hana Financial Network (for Small & Mid-sized) Homepage Platform Service Integration] (Hana Cloudia, ’20. 09 ~ ’20. 11)
- Project Goal: Support for the establishment and transition of the Paas platform for small and medium-sized affiliates in Hana Financial Group
- Responsibilities: Platform design and deployment
   (1) Multi-Master Replication AP/DB configuration using Docker Container to minimize cost.
   (2) Configure the Devops environment to automatically build/deploy pipeline when committing the source of the external homepage developer is changed.

[Hana Members System Cloud Migration] (Hana Members, ’20. 04 ~ ’20. 07)
- Project Goal: Hana Members System Cloud migrate to Oracle Cloud / Hana Cloudia Project
- Responsibilities: AA
   (1) Migrate Hana Members System to Oracle Cloud and Hana Cloud
   (2) Identify issues related to configuring and relocation of applications and support decision making to resolve issues

[Hana Savings Bank Contactless Platform Integration] (Hana Savings Bank, ’20. 02 ~ ’20. 08)
- Project Goal: Banking Contactless Platform (Internet, Mobile Banking ETC) Integration
- Responsibilities: AA / Technical support for open source solution
   (1) Internet, Mobile Banking System (ASP outsourcing) Migration/Integration to Hana Cloudia
   (2) Restful API system design and configuration with Netflix OSS (Zuul, Eureka, Ribbon ETC) and Docker container
   (3) Log management using ELK. Stateless Session configuration using Redis Cluster (Multi-Master).

[Hana Financial Group Data Governance Integration TFT] (Hana Financial Group, ’19. 11 ~ ’20. 02)
- Project Goal: Design data governance and data lake and pipeline for Hana financial Group
- Responsibilities: AA / Technical support for open source solution
   (1) Conceptual design private Data Lake for Hana Group Information Integration and Realtime / batch data pipeline flow architecture using Kafka and ETL
   (2) Review of strategies for establishing integrated data governance of affiliates organized by Hana Financial Group
   (3) Configuring HCIMS Advanced Design and Architecture for Data Transition Platforms between Hana affiliated companies

[Hana Savings Bank Transformation] (Hana Savings Bank, ’19. 06 ~ ’19. 10)
- Project Goal: Banking Core Systems Rebuild & Transformation for Next Generation
- Responsibilities: AA
   (1) Banking core system rebuild and transformation using MDD, Devon, XFrame UI
   (2) Reviewing the system transfer configuration and design of the development environment to the Savings Bank with Hana bank global new system framework
   (3) Compatibility review of response development architecture for applications
